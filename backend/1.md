# Backend Talent API - .NET 10

## Qué hicimos

CRUD completo de **Usuarios** con arquitectura en capas, conectado a **Supabase (PostgreSQL)**.

## Arquitectura

```
Controller → Service → Repository → Data (DbContext) → PostgreSQL
```

| Carpeta | Responsabilidad |
|---|---|
| `Controllers/` | Recibe peticiones HTTP (endpoints) |
| `Services/` | Lógica de negocio (hasheo de contraseñas, validaciones) |
| `Repository/` | Consultas a la base de datos |
| `Data/` | Configuración de Entity Framework (AppDbContext) |
| `Entities/` | Modelos que representan tablas (Usuario) |
| `DTO/` | Objetos para enviar/recibir datos del frontend |

## Endpoints actuales

| Método | URL | Qué hace |
|---|---|---|
| GET | `/api/usuarios` | Trae todos los usuarios |
| GET | `/api/usuarios/{id}` | Trae un usuario por ID |
| POST | `/api/usuarios` | Crea un usuario |
| PUT | `/api/usuarios/{id}` | Actualiza un usuario |
| DELETE | `/api/usuarios/{id}` | Elimina un usuario |

## Paquetes NuGet instalados

- `Swashbuckle.AspNetCore` → Swagger UI
- `Npgsql.EntityFrameworkCore.PostgreSQL` → Conexión a PostgreSQL
- `BCrypt.Net-Next` → Hasheo de contraseñas
- `Microsoft.AspNetCore.OpenApi` → OpenAPI

## Setup para compañeros

1. Clonar el repo
2. Abrir `Talent.slnx` en Visual Studio
3. Los paquetes NuGet se restauran automáticamente
4. Crear el archivo `Talent.API/appsettings.Development.json` con:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=TU_HOST;Database=postgres;Username=postgres;Password=TU_PASSWORD;SSL Mode=Require;Trust Server Certificate=true"
  }
}
```

5. Darle Play → Swagger en `https://localhost:7016/swagger`
