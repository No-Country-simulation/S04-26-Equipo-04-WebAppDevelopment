# PRD MVP v1 - Empleabilidad +45 basada en evidencia

## 1) Objetivo del producto

Construir un MVP que conecte de forma efectiva talento +45 con oportunidades reales, mediante formación práctica y evidencia verificable de habilidades.

## 2) Problema que se resuelve (sin desviaciones)

### Problema principal

Existe una desconexión entre:

1. Talento +45 con experiencia sólida.
2. Formación relevante y aplicada.
3. Visibilidad efectiva frente a empresas.

### Señales del problema

- Formación actual demasiado teórica.
- Dificultad para demostrar valor actualizado.
- Baja visibilidad de perfiles senior.
- Empresas sin acceso simple a talento +45 validado.

## 3) Segmento objetivo (MVP)

Profesionales de 40 a 54 años en transición laboral, con experiencia en:

- liderazgo medio/gestión, o
- roles técnicos/analíticos.

Ámbito inicial: comunidad en Argentina (CABA y Buenos Aires).

## 4) Propuesta de valor del MVP

- Diagnóstico de empleabilidad accionable.
- Ruta de actualización corta y personalizada.
- Perfil profesional dinámico con evidencia.
- Marketplace para preselección por empresas.

## 5) Alcance funcional (solo lo esencial)

### 5.1 Diagnóstico inicial

- Evaluación breve de habilidades (digitales, cognitivas y socioemocionales).
- Resultado: brechas concretas + recomendación de ruta.

### 5.2 Ruta de aprendizaje personalizada

- 3 a 5 módulos prácticos.
- Seguimiento de progreso por módulo.
- Hito de finalización para desbloquear visibilidad.

### 5.3 Perfil profesional dinámico (CV vivo)

- Experiencia previa estructurada.
- Skills desarrolladas en plataforma.
- Evidencia (logros/módulos completados).

### 5.4 Talent Marketplace (versión base)

- Empresas visualizan perfiles.
- Filtros por skill, trayectoria y avance.
- Preselección + feedback estructurado.

## 6) Fuera de alcance (fase posterior)

- Matching avanzado con IA compleja.
- Integraciones profundas con ATS/LMS externos.
- Cobertura masiva de todos los perfiles +45.
- Gamificación avanzada.

## 7) Historias de usuario clave

### Usuario profesional

1. Como profesional en transición, quiero un diagnóstico claro para saber qué actualizar primero.
2. Como profesional, quiero una ruta corta y práctica para mejorar mi empleabilidad.
3. Como profesional, quiero un perfil dinámico para mostrar evidencia actual y no solo historial.

### Usuario empresa

4. Como reclutador, quiero filtrar talento senior validado para reducir riesgo de contratación.
5. Como reclutador, quiero dejar feedback estructurado para acelerar decisiones.

## 8) Criterios de aceptación del MVP

- El usuario completa diagnóstico en menos de 15 minutos.
- Se genera una ruta personalizada inmediatamente.
- El usuario puede completar módulos y visualizar progreso.
- El perfil dinámico refleja evidencia de avances.
- La empresa puede filtrar y preseleccionar perfiles.
- Se registra feedback de empresa sobre candidatos.

## 9) Métricas de éxito (problema-solution fit)

- % de usuarios que completan diagnóstico.
- % de usuarios que completan ruta mínima.
- % de perfiles con evidencia publicada.
- Tasa de preselección por empresas.
- Tiempo medio a primera oportunidad concreta.

## 10) Riesgos y mitigaciones

- Riesgo: producto demasiado amplio.
  - Mitigación: segmentación estricta del MVP.
- Riesgo: baja finalización de módulos.
  - Mitigación: rutas cortas, enfoque práctico y seguimiento simple.
- Riesgo: poca tracción empresarial.
  - Mitigación: activar primero empresas aliadas y ONG asociada.

## 11) Roadmap sugerido (8 semanas)

- Semanas 1-2: Diagnóstico + onboarding.
- Semanas 3-4: Ruta de aprendizaje + tracking.
- Semanas 5-6: Perfil dinámico (CV vivo).
- Semanas 7-8: Marketplace base + feedback.

## 12) Regla de foco de producto

Cada funcionalidad nueva debe responder explícitamente:

1. ¿Mejora empleabilidad real?
2. ¿Aumenta evidencia verificable?
3. ¿Mejora conexión con oportunidades?

Si no responde al menos a una, no entra en MVP.
